# Zuoyanblogs

Zuoyanblogs ブログプロジェクトは、オープンソースプロジェクトである Mizuki テーマフレームワークの改良版です。Astro をベースに構築された、モダンで機能豊富な静的ブログテンプレートで、高度な機能と美しいデザインを誇ります。

- **統合設定アーキテクチャ:** 動的なコンポーネント管理とシーケンシャル設定をサポートする、まったく新しいモジュール型のコンポーネント設定システムです。

## ⚡ このプロジェクトの実行方法

1. **リポジトリのクローンを作成:**

```bash

git clone https://github.com/Zuoyan233/Zuoyanblogs.git

cd Zuoyanblogs

```

2. **依存関係をインストール:**

```bash
# pnpm がインストールされていない場合は、まずインストールしてください
npm install -g pnpm

# プロジェクトの依存関係をインストール
pnpm install

```

3. **ブログの設定:**

- `src/config.ts` を編集してブログの設定をカスタマイズします。
- サイト情報、テーマカラー、バナー画像、ソーシャルリンクを更新します。
- おすすめページの機能を設定します。
- (オプション) コンテンツリポジトリの分離を設定します - 詳細については、Mizuki ドキュメントの [コンテンツリポジトリの設定](https://docs.mizuki.myqil.com/Other/separation/) をご覧ください。

4. **おすすめページの設定**
- **アニメ追跡ページ:** `src/pages/anime.astro` 内のアニメリストを編集します。
- **友達リンクページ:** `src/content/spec/friends.md` 内の友達データを編集します。
- **アルバムページ:** `public/images/albums` 内のアルバム情報を編集します，使い方については、[アルバム機能ユーザーガイド](./public/images/albums/README.md) を参照してください。
- **マイデバイスージ:** `src/data/devices.ts` 内のデバイス情報を編集します。
- **日記ページ:** `src/data/diary.ts` 内の投稿を編集します。
- **Aboutページ:** `src/content/spec/about.md` 内のコンテンツを編集します。
- **スポンサーシップページ:** `src/content/spec/sponsors.md` のコンテンツを編集します。
  - `src/config.ts` で、支払い用QRコードを設定する `addpaymentConfig` を探します，支払い用QRコードは `public/images/sponsors` に保存されています。
- **フィードバックページ:** `src/content/spec/feedback.md` のコンテンツを編集し。
  - `src/congfig.ts` で `contactEmailConfig` を見つけて、Web サイト管理者の電子メール アドレスを構成します。
  - `public/images/albums/contact` に管理者のSNS連絡先を追加してください。
- **プロジェクト表示ページ：** `src/data/projects.ts` で表示内容を編集します。
- **スキル表示ページ：** `src/data/skills.ts` で表示内容を編集します。
- **タイムラインページ：** `src/data/timeline.ts` で表示内容を編集します。

5. **記事コンテンツ管理**
- **新しい投稿を作成:** `pnpm new-post <filename>`
- **投稿を編集:** `src/content/posts/` 内のファイルを変更します。
- **ページのカスタマイズ:** `src/content/spec/` 内の特別ページを編集します。
- **画像を追加:** `src/assets/` または `public/` 内に画像を配置します。
- **Markdown 拡張構文:** 詳細については、Mizuki ドキュメントの [Markdown 拡張構文](https://docs.mizuki.mysqil.com/press/Markdown/customize/) をご覧ください。

Frontmatter フィールドの説明:

- **title**: 記事のタイトル (必須)
- **published**: 公開日 (必須)
- **description**: 記事の説明 (SEO とプレビュー用)
- **image**: 表紙画像のパス (記事ファイルからの相対パス)
- **tags**: 分類用のタグの配列
- **category**: 記事のカテゴリ
- **draft**: 本番環境で記事を非表示にするには `true` に設定
- **pinned**: 記事をピン留めするには `true` に設定
- **lang**: 記事の言語 (サイトのデフォルト言語と異なる場合のみ設定)

6. **開発サーバーを起動します:**

```bash
pnpm dev
```

ブログは `http://localhost:4321` で利用できます。

7. **すべてのコマンドはプロジェクトのルートディレクトリで実行されます:**

| コマンド                   | アクション                             |
|:---------------------------|:---------------------------------------|
| `pnpm install`             | 依存関係をインストールします           |
| `pnpm dev`                 | `localhost:4321` でローカル開発サーバーを起動します |
| `pnpm build`               | 本番環境サイトを `./dist/` にビルドします |
| `pnpm preview`             | デプロイ前にローカルでビルドをプレビューします |
| `pnpm check`               | Astro エラーチェックを実行します |
| `pnpm format`              | Biome を使用してコードをフォーマットします |
| `pnpm lint`                | コードの問題をチェックして修正します |
| `pnpm new-post <filename>`   | 新しいブログ投稿を作成します  |
| `pnpm astro ...`           | Astro CLI コマンドを実行します |

開発者の皆様によりスムーズで使い慣れたエクスペリエンスを提供するために、コードのフォーマットと修正には引き続き Biome を使用しています。

Mizuki バージョン 7.6 以降では Prettier + ESLint に切り替わりましたが、Biome の「すぐに使える」機能は、追加の設定コストを回避でき、私たちのワークフローにより適していると考えています。
